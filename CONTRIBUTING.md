# 🤝 Contributing to NexusCore-SLIATE

We are thrilled that you want to push code to NexusCore! To keep our projects scaling systematically and cleanly, all active members must adhere to this unified workflow.

## 🛠️ Git & Branching Standards
1. Do not commit directly to the `main` branch. 
2. Fork or create a descriptive branch from `main` using standard prefixes:
   * `feature/your-feature-name`
   * `bugfix/issue-description`
   * `docs/update-documentation`
3. Open a Pull Request (PR) targeting `main` and link it to an active issue tracker item.

## 📝 Commit Conventions
We use strict **Conventional Commits** to keep our logs professional:
* `feat: add dark-mode utility to navigation`
* `fix: correct variable leak in scanner script`
* `docs: update deployment environment configurations`

## 👥 Peer Code Review Protocol
* Every PR requires at least **one approved review** from your respective Track Lead before merging.
* Be supportive, offer constructive guidance during reviews, and learn through collaboration!
